# План работы и разработки для семейного проекта

## Этапы разработки и работы

### 1. Анализ требований и проектирование

- **Определение функций и задач**:
  - Создание и управление данными о членах семьи.
  - Извлечение биографической информации.
  - Вывод связанных данных через Telegram-бота.
  - Использование ИИ для анализа и ответа.

- **Структура базы данных**:
  - Создание ER-диаграммы для базы данных PostgreSQL.
  - Таблица `Relatives` (Родственники) с полями: `Id`, `Имя`, `Фамилия`, `Дата рождения`, `Биография`, `Степень родства`, и т.д.
  - Таблица `Relationships` для описания связей между родственниками.

### 2. Настройка окружения

- **База данных PostgreSQL**:
  - Развернуть и настроить PostgreSQL.
  - Создать структуру таблиц, необходимую для хранения данных о родственниках.

- **ASP.NET Core API**:
  - Создать ASP.NET Core проект.
  - Настроить доступ к базе данных с использованием Entity Framework Core.

- **Telegram-бот**:
  - Зарегистрировать Telegram-бота через BotFather.
  - Получить токен доступа для API.

- **Интеграция ИИ**:
  - Настроить взаимодействие с API GPT-4.1.

### 3. Разработка API

- **CRUD операции**:
  - Реализовать ключевые API методы: создание, чтение, обновление, удаление информации о родственниках.

- **API для бота**:
  - Функции для ответа на запросы от Telegram бота с использованием данных из базы.

### 4. Разработка Telegram-бота

- **Логика команды**:
  - Команды для начала (`/start`), добавления, поиска, вывода информации (`/add`, `/info <name>`, `/relationship <name>`).

- **Обработка запросов**:
  - Настроить бота на отправку запросов к ASP.NET Core API.

### 5. Интеграция с GPT-4.1

- **Настройка GPT-4.1** для обработки вопросов о биографиях.
- **Тестирование** ответов и корректировка их информации.

### 6. Тестирование

- Тестирование Telegram-бота и API.
- Проверка корректности взаимодействия с базой данных и корректности данных.

### 7. Развертывание

- Развертывание API на удаленном сервере (например, через Docker или Azure).
- Настройка бота Telegram для работы в production-среде.

### 8. Поддержка и улучшения

- Сбор и анализ пользовательских отзывов.
- Дополнение функциональности по запросам пользователей.

## Заключение

Этот план охватывает ключевые задачи при разработке такой системы. Разработка проекта требует внимания как к техническим аспектам (структура БД, взаимодействие с API), так и к взаимодействию с пользователем через бота. Не забывайте об учёте техники безопасности данных, особенно если храните личные данные пользователей.
